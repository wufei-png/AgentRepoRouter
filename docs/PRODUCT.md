# 产品规划

> 本文档描述的是 **未来规划**，不是当前已经实现的功能清单。当前落地实现请以 `scripts/install.sh`、`skills/router/SKILL.*.md` 和测试覆盖为准。

## 方向

OrchAI 的长期目标，是在当前 OpenClaw Router Skill 的基础上，逐步演进成一个多项目、多 agent 的本地编程助理编排层。

## 当前基线

当前仓库已经实现的范围：

- 安装脚本和环境检查
- 多 CLI 选择
- 项目发现与 `repo_mappings.json` 生成
- Router Skill 部署
- 基于 repo 和任务描述的运行时路由约定

## 未来规划

### 1. 更强的统一入口

- 更好的会话入口和任务发起体验
- 更清晰的项目与 CLI 选择反馈
- 更完善的运行状态可视化

### 2. 更可靠的智能路由

- 提升 repo 识别准确率
- 提升 agent 选择准确率
- 引入更明确的 fallback 和冲突处理策略

### 3. 更丰富的项目级扩展

- 更规范的项目级 skill / agent 约定
- 更清晰的全局配置与项目配置边界
- 更好的跨 CLI 配置映射

### 4. 知识与外部系统集成

- GitHub / GitLab 相关能力接入
- 文档检索和知识库能力接入
- 代码索引和上下文增强

### 5. 长任务与自动化

- 批量审查和批量执行任务
- 定时任务和周期性任务
- 长任务状态跟踪和结果汇总

## 阶段建议

### Phase A

- 稳定当前安装与路由体验
- 补齐文档和测试
- 收紧跨 CLI 约定

### Phase B

- 增强多项目规则
- 增强项目级扩展能力
- 补齐更多真实场景验证

### Phase C

- 接入外部知识与平台能力
- 设计长任务调度模型
- 引入更完整的任务可观测性

## 成功指标

- 路由行为可预测且易解释
- CLI 选择与 repo 选择错误率持续下降
- 主文档和实现保持同步
- 新项目接入成本足够低
